共享命名约定允许团队高效地协作。此规则检查所有函数名是否与提供的正则表达式匹配。

不合规的代码示例

使用默认提供的正则表达式： ^[a-z][a-zA-Z0-9]*$:

public int DoSomething(){...}

合规的解决方案

public int doSomething(){...}

例外

方法伴随一个@inheritdoc注释， 以及魔法方法 (__construct(), __destruct(), __call(), __callStatic(), __get(), __set(), __isset(), __unset(), __sleep(), __wakeup(), __toString(), __invoke(), __set_state(), __clone(), __debugInfo()) 将被忽视。

function __construct(){...}
function __destruct(){...}

/**
 * {@inheritdoc}
 */
function myFunc(){...}
