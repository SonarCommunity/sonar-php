子表达式中的赋值很难发现，会降低代码的可读性。理想情况下，子表达式不应该有副作用。

不合规的代码示例

if ($val = value() && check()) { // 不合规
}

合规的解决方案

$val = value();
if ($val && check()) {
}

或者

if ($val == value() && check()) { // 也许事实上，赋值运算符是意料之中的
}

例外

在while 语句环境中赋值和在关系表达式中赋值是允许的。

while (($line = next_line()) != NULL) {...}

while ($line = next_line()) {...}

请参阅

  • MITRE, CWE-481 - Assigning instead of Comparing
  • CERT, EXP45-C. - Do not perform assignments in selection statements
  • CERT, EXP51-J. - Do not perform assignments in conditional expressions
