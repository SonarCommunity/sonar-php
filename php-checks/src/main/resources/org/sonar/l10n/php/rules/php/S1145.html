伴随条件的if 语句条件总是为false会有影响，使代码块无效。伴随条件的if 语句条件总是为true会完全冗余，并且会降低代码的可读性。

出现这种代码有三个可能的原因：

  • 调试期间更改了if语句，并且调试代码被提交。
  • 有些值没有设置。
  • 有些逻辑并没有做程序员认为它该做的事情。

如果出现上述情况，无条件的if语句应该被删除。

不合规的代码示例

if (true) {  // 不合规
  doSomething();
}
...
if (false) {  // 不合规
  doSomethingElse();
}

合规的解决方案

doSomething();

  • MITRE, CWE-570 - Expression is Always False

  • MISRA C:2012, 14.3 - Controlling expressions shall not be invariant
