重写一个方法仅仅只是从超类调用同一个方法，而不执行任何其他操作，这是无用的和误导性的。唯一合理的一次是在 final 重写方法中，其效果是锁定父类行为。 该规则忽略了对equals, hashCode 和 toString的重写。

不合规的代码示例

class Child extends Parent {

  public function func($n,$m) {
    parent::func($n$m);  // 不合规
  }
}

class Parent {
  public function func($n, $m) {
    // 做点什么（do something）
  }
}

合规的解决方案

class Child extends Parent {

  public function func($n,$m) {
    parent::func($n$m);
    // 做额外的事情...
  }
}

class Parent {
  public function func($n, $m) {
    // 做点什么（do something）
  }
}

或者

class Child extends Parent {
  // 函数被消除
}

class Parent {
  public function func($n, $m) {
    // 做点什么 （do something）
  }
}
