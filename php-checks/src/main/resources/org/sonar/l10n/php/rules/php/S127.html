一个for循环停止条件的应该针对一个不变值(即在每个循环迭代的开始和结束时都为true的值)测试循环计数器。理想情况下，这意味着在循环开始之前将停止条件设置为局部变量。

如果停止条件不是不变的，那么效率会稍微低一些，并且很难理解和维护，并且可能会在将来引入错误。

此规则跟踪三种类型的非不变停止条件：

  • 当循环计数器在for循环的主体中更新
  • 当停止条件依赖于方法调用
  • 当停止条件依赖于对象属性，这些属性可能在循环执行期间更改。

不合规的代码示例

for ($i = 0; $i < 10; $i++) {
  echo $i;
  if(condition) {
    $i = 20;
  }
}

合规的解决方案

for ($i = 0; $i < 10; $i++) {
  echo $i;
}

请参阅

  • MISRA C:2004, 13.6 - Numeric variables being used within a for loop for iteration counting shall not be modified in the body of the loop.
  • MISRA C++:2008, 6-5-3 - The loop-counter shall not be modified within condition or statement.
