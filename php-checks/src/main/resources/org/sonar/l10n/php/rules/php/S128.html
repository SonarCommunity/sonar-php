当执行没有明确地终止于 switch case 的末尾时，它将继续执行下列用例的语句。虽然有时这是故意的，但这往往是一个错误，导致意想不到的行为。

不合规的代码示例

switch ($myVariable) {
  case 1:
    foo();
    break;
  case 2:  // 同时执行“doSomething()”和“doSomethingElse()”。这是故意的吗?
    do_something();
  default:
    do_something_else();
   break;
}

合规的解决方案

switch ($myVariable) {
  case 1:
    foo();
    break;
  case 2:
    do_something();
    break;
  default:
    do_something_else();
   break;
}

例外

在以下情况下可放宽此规则：

switch ($myVariable) {
  case 0:                  // 空Case，用于为一组用例指定相同的行为。
  case 1:
    do_something();
    break;
  case 2:                  // 继续语句的使用
    continue;
  case 3:                  // Case包含一个跳转语句(exit, return, break &etc)
    exit(0);
  case 4:
    echo 'Second case, which falls through';
    // 不跳出       <- 注释用于在非空的案例体中有意跳过
  default:                 // 对于最后一种情况，可以选择使用break语句
    doSomethingElse();
}
