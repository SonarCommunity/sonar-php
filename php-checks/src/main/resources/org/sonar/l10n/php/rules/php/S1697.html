当null测试中的相等运算符或它后面的逻辑运算符被反转时，代码的外观是在解除引用对象之前会对其进行安全的null测试。不幸的是，结果正好相反————对象被null测试， 并且如果是null的话，解除引用only，从而导致一个被保证的空指针解除引用。

不合规的代码示例

if ($obj == null && $obj->isOpen()) {
  echo "Object is open";
}

if ($obj != null || $obj->isOpen()) {
  echo "Object is not open";
}

合规的解决方案

if ($obj == null || $obj->isOpen()) {
  echo "Object is open";
}

if ($obj != null && !$obj->isOpen()) {
  echo "Object is not open";
}
