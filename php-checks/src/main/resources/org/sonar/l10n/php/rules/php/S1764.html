在二进制运算符的两边使用相同的值几乎总是一个错误。对于逻辑操作符，它要么是复制/粘贴错误，就是是一个bug，要么只是浪费代码，应该加以简化。 在有位操作符和大多数二进制数学操作符的情况，在操作符的两边都有相同的值会产生可预测的结果，应该进行简化。

不合规的代码示例

if ( $a == $a ) { // 总是为 true
  doZ();
}
if ( $a != $a ) { // 总是为 false
  doY();
}
if ( $a == $b && $a == $b ) { // 如果第一个判断为 true，第二个也是
  doX();
}
if ( $a == $b || $a == $b ) { // 如果第一个判断为 true，第二个也是
  doW();
}

$j = 5 / 5; //always 1
$k = 5 - 5; //always 0

例外

左移1到1在位掩码的构造中很常见，但是被忽略了。

$i = 1 << 1; // 合规
$j = $a << $a; // 不合规
