PHP4中使用 var关键字声明变量的方法在PHP5的早期版本中是不提倡的。尽管在最近的版本中它并没有被认为是不受欢迎的，但它仍然不是最佳的使用方法。 当 var 出现时，它被解释为 public 的同义词，并按此处理。应该使用 public 代替。

来自PHP手册：

出于兼容性的原因(作为public关键字的同义词)，仍然支持使用var关键字声明变量的PHP4方法。 在5.1.3之前的PHP 5中，使用它会生成一个E_STRICT警告。

不合规的代码示例

<?php
class Foo
{
    var $bar = 1;
}

合规的解决方案

<?php
class Foo
{
    public $bar = 1;
}
