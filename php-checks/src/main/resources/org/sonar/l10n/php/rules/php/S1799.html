exit(...) 和 die(...) 语句绝对不应该在Web PHP页面中使用，因为这可能会导致非常糟糕的用户体验。在这种情况下，最终用户可能会觉得web站点已经关闭或遇到了致命错误。

当然，PHP也可以用来开发命令行应用程序，在这种情况下，使用exit(...) 或者 die(...)语句是合理的，但必须保持有限，不能在应用程序中到处使用。 我们希望异常来处理错误，并且在这些异常离开应用程序之前捕获，以便在 exit(...) 或者 die(...)语句的帮助下指定退出代码。

不合规的代码示例

class Foo {
    public function bar($param)  {
        if ($param === 42) {
            exit(23);
        }
    }
}

合规的解决方案

class Foo {
    public function bar($param)  {
        if ($param === 42) {
            throw new Exception('Value 42 is not expected.');
        }
    }
}
