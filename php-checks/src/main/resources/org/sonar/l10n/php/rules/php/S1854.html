当为局部变量分配了一个后续指令无法读取的值时，就会产生无作用储存体。计算或检索一个值，仅仅是覆盖或丢弃它，可能意味着代码中存在严重错误。 虽然这不是一个错误，充其量也只是浪费资源。因此，应使用所有计算值。

不合规的代码示例

$i = $a + $b; // 不合规；覆盖值之前未使用计算结果
$i = compute();

合规的解决方案

$i = $a + $b;
$i += compute();

例外

这个规则忽略了对 -1，0，1，null，true， false， ""， [] 和 array()的初始化。

请参阅

  • MITRE, CWE-563 - Assignment to Variable without Use ('Unused Variable')
  • CERT, MSC13-C. - Detect and remove unused values
  • CERT, MSC56-J. - Detect and remove superfluous code and values
