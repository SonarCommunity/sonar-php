传递对函数参数的引用意味着方法对参数所做的任何修改也将对原始值进行修改，因为引用具有将两个变量指向同一内存空间的效果。 这个特性很难正确地使用，特别是当被调用者不期望引用时，而且函数调用中引用使用不当会降低代码的效率，而不是提高代码的效率。

此外，根据PHP手册:

从PHP 5.3.0开始，会收到一个警告，“call-time pass-by-reference”，这是不推荐的……从PHP5.4.0开始，”call-time pass-by-reference“被删除了，因此使用它将引发致命错误。

不合规的代码示例

myfun(&$name);  // 不合规

合规的解决方案

myfun($name);
