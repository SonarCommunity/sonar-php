定义和使用全局变量和全局函数时，如果该约定指示OOP，则由于多种原因，可能会造成混淆，并且难以正确使用：

  • 有名字冲突的风险。
  • 全局函数必须是无状态的，否则会导致难以跟踪的bug。
  • 全局变量可以从任何地方更新，并且可能不再保存您期望的值。
  • 正确地测试使用全局函数的类是困难的。

这些变量和函数不应该被全局声明，而是应该被移动到一个类中，可能标记为 static，这样它们就可以在没有类实例的情况下使用。

此规则检查是否只使用面向对象的编程，以及是否在类外部声明任何函数或过程。

不合规的代码示例

<?php

$name = "Bob"; // 不合规

function doSomething($arg) {   // 不合规
  //...
}

class MyClass {
    //...
}

合规的解决方案

<?php
class MyClass {

  public static $name = "Bob"; // 合规

  public static function doSomething($arg) {              // 合规
    //...
  }
  //...
}
