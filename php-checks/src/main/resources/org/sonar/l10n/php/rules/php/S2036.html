定义类和变量等符号的文件可以包含在许多文件中。简单地执行包含应该不会对这些文件产生任何影响，除了声明新的符号。例如，包含类定义的文件不应该包含副作用，比如 print 语句，这些语句将在包含时自动评估。逻辑应分为仅符号文件和仅副作用文件。符号定义文件中不允许的操作类型包括但不限于：

  • 生成输出
  • 修改 ini 设置
  • 发出错误或异常
  • 修改全局变量或静态（static）变量
  • 读/写文件

不合规的代码示例

<?php

print "Include worked!";

class foo {
  // ...
}

合规的解决方案

<?php

class foo {

  public function log() {
    print "Include worked!";
  }

}
