MD5算法及其后续的SHA-1不再被认为是安全的，因为很容易产生哈希冲突。也就是说，生成相同的MD5或SHA-1哈希的不同输入只需要很少的计算工作，使用新的、相同的哈希值使攻击者具有与原始哈希值相同的访问权限。 这也适用于其他消息摘要算法: MD2，MD4，MD6，HAVAL-128，HMAC-MD5，DSA(使用SHA-1)，RIPEMD，RIPEMD-128，RIPEMD-160，HMACRIPEMD160。

考虑使用更安全的替代方案，如SHA-256或SHA-3。

不合规的代码示例

$password = ...

if (md5($password) === '1f3870be274f6c49b3e31a0c6728957f') { // 不合规；md5()哈希算法对密码管理不安全
   [...]
}

if (sha1($password) === 'd0be2dc421be4fcd0172e5afceea3970e2f3d940') { // 不合规；sha1()哈希算法对密码管理不安全
   [...]
}
