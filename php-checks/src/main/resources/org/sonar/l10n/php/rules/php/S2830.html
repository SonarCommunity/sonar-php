依赖注入是一种软件设计模式，在这种模式中，一个或多个依赖项(或服务)被注入或通过引用传递到一个依赖对象(或客户机)中，并成为客户机状态的一部分。 该模式将客户机依赖关系的创建与其自身行为分离开来，从而允许程序设计松散耦合，并遵循依赖关系倒置和单一责任原则。

不合规的代码示例

class SomeClass {

  public function __construct() {
    $this->object = new SomeOtherClass();  // 不合规
  }
}

合规的解决方案

class SomeClass {

  public function __construct(SomeOtherClass $object) {
    $this->object = $object;
  }
}
