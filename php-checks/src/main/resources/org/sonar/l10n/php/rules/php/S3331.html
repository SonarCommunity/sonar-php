cookie的域指定哪些网站应该能够读取它。如果保留空白，浏览器应该只将cookie发送到与发送域完全匹配的站点。例如，如果cookie 是由lovely.dream.com设置的，那么它应该只由该域来读取，而不是由 nightmare.com 或甚至 strange.dream.com读取。如果希望允许子域访问cookie，可以通过在cookie的域前面添加一个点来指定它，例如: .dream.com。但是cookie域应该始终至少使用两个级别。

可以通过编程或配置来设置cookie域。当任何cookie域设置为单一级别时，正如 .com，此规则会引发一个问题。

不合规的代码示例

setcookie("TestCookie", $value, time()+3600, "/~path/", ".com", 1); // 不合规
session_set_cookie_params(3600, "/~path/", ".com"); // 不合规

// 在php.ini内部
session.cookie_domain=".com"; // 不合规

合规的解决方案

setcookie("TestCookie", $value, time()+3600, "/~path/", ".myDomain.com", 1);
session_set_cookie_params(3600, "/~path/", ".myDomain.com");

// 在php.ini内部
session.cookie_domain=".myDomain.com";
