allow_url_fopen 和 allow_url_include允许从URL的脚本中读取代码。从站点外部吸收可执行代码的能力，加上不完善的输入清理，可能会使攻击者暴露你的站点。 即使你的输入过滤今天是完美的，你准备好打赌你的网站在未来总是完美的吗？

当在php.ini 中显式启用这两个属性时，以及当 allow_url_fopen(默认为启用)未显式禁用时，该规则会引发一个问题。

不合规的代码示例

; 在php.ini中，不合规；allow_url_fopen 未显式禁用
allow_url_include=1   ;不合规

合规的解决方案

; 在php.ini中
allow_url_fopen=0
allow_url_include=0
