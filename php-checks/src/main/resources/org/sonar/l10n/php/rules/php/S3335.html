默认情况下，cgi.force_redirect php.ini 配置是打开的，它防止在PHP作为CGI运行时对脚本进行未经身份验证的访问。不幸的是，它必须在IIS、OmniHTTPD和Xitami上禁用，但在其他所有情况下都应该启用。

当显示禁用cgi.force_redirect 时，该规则会引发一个问题。

不合规的代码示例

; 在php.ini中
cgi.force_redirect=0  ; 不合规

请参阅

  • OWASP Top 10 2017 Category A6 - Security Misconfiguration
  • MITRE, CWE-305 - Authentication Bypass by Primary Weakness
