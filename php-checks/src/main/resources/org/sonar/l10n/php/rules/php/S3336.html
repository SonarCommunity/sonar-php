当禁用cookie时，PHP的session.use_trans_sid自动将用户的会话id附加到urls。从表面上看，这似乎是一种很好的方式，让uncookie-able用户使用你的网站。 实际上，它使这些用户的会话很容易被任何可能的人劫持：

  • 查看用户身后的URL
  • 由用户发送URL
  • 从浏览器历史记录中检索URL
  • ......

因此，更好的做法是对用户进行一点强制要求，强迫他们打开cookie。

由于session.use_trans_sid 在默认情况下是关闭的，所以当显式启用时，该规则会引发一个问题。

不合规的代码示例

; 在php.ini文件
session.use_trans_sid=1  ; 不合规
