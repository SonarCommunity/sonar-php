LDAP客户端使用“绑定请求”向LDAP服务器进行身份验证，该“绑定请求”提供了简单的身份验证方法。

LDAP中的简单身份验证可以与三种不同的机制一起使用：

  • 通过使用用户名和密码长度为零的绑定请求执行匿名身份验证机制。
  • 通过执行密码长度为零的绑定请求，实现未经身份验证的身份验证机制。
  • 名称/密码验证机制，方法是使用非零长度的密码值执行绑定请求。

匿名绑定和未经身份验证的绑定允许在不提供密码的情况下访问LDAP目录中的信息，因此强烈建议不要使用它们。

不合规示例

$ldapconn = ldap_connect("ldap.example.com");

if ($ldapconn) {
    $ldapbind = ldap_bind($ldapconn); // 不合规； 匿名身份验证，未提供用户名/密码
}

合规的方案

$ldaprdn  = 'uname';
$ldappass = 'password';

$ldapconn = ldap_connect("ldap.example.com");

if ($ldapconn) {
    $ldapbind = ldap_bind($ldapconn, $ldaprdn, $ldappass); // 合规
}
