信令进程是安全敏感的。它在过去导致了以下漏洞：

  • CVE-2009-0390
  • CVE-2002-0839
  • CVE-2008-1671

在没有正确检查哪个进程将接收到信号的情况下发送信号可能导致拒绝服务。

检查是否有下列问题：

  • 将信号发送到的进程的PID来自一个不可信的源。例如，它可以来自一个可写的文件。
  • 请求发送信号的用户可能没有发送这些信号的权限。

如果对其中任何一个问题的回答是肯定的，那么就有风险。

推荐的安全编码实践

  • 如果信号是由于用户的请求而发送的。检查用户是否允许发送此信号。例如，如果用户不拥有进程，您可以禁止它。
  • 保护读取进程PID的源。
  • 运行以最小权限发送信号的进程。

可疑的代码示例

posix_kill(42, 42); // 可疑的
