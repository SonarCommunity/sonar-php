PHP 5.3向该语言引入了名称空间。这种机制的使用应该优于 include 或 include_once 或 require 或 require_once，因为它解决了两个常见的问题：

  • 它避免了名称冲突
  • 它提供了创建别名的能力，从而提高了代码的可读性

Drupal的第8版开始，Drupal就依赖名称空间来符合 PSR-4标准。Drupal的模块应该符合PSR-4标准，因此不应该再依赖于 include 或 include_once 或 require 或 require_once 函数。

不合规的代码示例

require_once('./modules/vegetable/src/Entity/Tomato.php');

合规的解决方案

use Drupal\vegetable\Entity\Tomato

例外

该规则不会对autoload.php 或 ScriptHandler.php 文件引发问题。
