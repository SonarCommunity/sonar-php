Dynamically setting session IDs with user-supplied data or insecure hashes may lead to session fixation attacks.

This may allow an attacker to hijack another user's session.

Recommended Secure Coding Practices

Don't manually generate session IDs, and use PHP's native functionality such as session_regenerate_id() instead.

If however you must generate your own IDs, use a cryptographically secure method, like bin2hex(random_bytes(16))

Questionable Code Example

session_id($_GET['my_id']);

Compliant Solution

session_regenerate_id();

// or
$sessionId = bin2hex(random_bytes(16));
session_id($sessionId);
